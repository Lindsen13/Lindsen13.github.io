Hosting a SQL Database for your Python projects with Digital Ocean

2020-09-18 • By Ivo

Using Digital Ocean's Droplets and MySQL to host a database for my personal projects

For some of my projects, I run a SQLite database, which is stored in a local file. It works most of the time as I want it, but in some occasions I could benefit from a more temporarily solution. For this, I would need a simple SQL database to store my data. I have 3 criteria:

  1. Allow to be hosted cheap
  2. Use an open-source SQL program, like Postgres or MySQL
  3. Option to expand in the future (and yes, perhaps with some extra costs)

I found a solution which will cost me just $5,- per month! That will suite my needs. I found this possibility at Digital ocean (Note, the link is an affiliate link. Clicking on it will help me hosting this website and my database). They have Droplets which are basically small virtual machines that allows us to host anything from web apps to databases. The latter is what we will do in this post.

Note, normally I love to geek around in a Python environment. For this project, we have to step a bit outside of our comfort zone and dive into Linux. Furthermore, I would like to mention that the steps shown below result in a database where I would not store any sensitive information. We basically allow everyone to send requests to the database. Not particular save you could say! Luckily, for my purpose it will do.

After it is done, we click the droplet and save the ipv4 address:

Now our droplet is initiated, and we can close down Digital ocean. The rest we will do from our local terminal.

Let's open a terminal and login to our virtual machine (Droplet). Note that you have to substitute your Droplet's IP for the ip address below.

ssh root@104.248.22.17

This will connect to the Droplet, after we have inserted our password. It might be that you have to type 'yes' to make a connection.

Great! Now we're in our Droplet. The first thing we have to do is to update the package index on our Droplet

sudo apt update

After this, we have to install MySQL on the Droplet. Make sure to answer 'Y' on the question(s) popping up.

sudo apt install mysql-server

Now we have to make sure we will be able to access MySQL from other machines in the future. For this, we have to make some changes in the mysqld.cnf file.

sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

This will open up a file, try to fint the following line (which should be somewhere below [mysqld])

bind-address            = 127.0.0.1

We will change it to

bind-address            = 0.0.0.0

We have to press ctrl+X to close down the file. Specify 'Y' to save, and hit enter.

Now MySQL will allow remote access, but the virtual machine won't. We have to poke a hole in the firewall, to allow remote connections to connect to port 3306 (where MySQL lives).

sudo ufw allow 3306

Now we can access both the Droplet and MySQL from a remote session. However, we do not have a user yet which can do this. Lets open up MySQL in the droplet to do so:

mysql

Here we run the following three commands: the first one will create a user with name 'pythoneverything' and password 'python123' We then give all privileges to this user, and finally 'flush' the privileges so they're in place. Note that '%' is a wildcard, which allows access from any IP address.

CREATE USER 'pythoneverything'@'%' IDENTIFIED BY 'python123';
GRANT ALL PRIVILEGES ON *.* TO 'pythoneverything'@'%' WITH GRANT OPTION;
FLUSH PRIVILEGES;

Now we can exit out of MySQL:

exit

And restart MySQL:

sudo systemctl restart mysql

Now we're done working on our Droplet, therefore we can exit out of the connection:

exit

Now we will be on a more familiar ground: We have to write some python code to test if we indeed can connect to the database. First, lets install the required dependency:

pip3 install mysql-connector-python

Next up, we need to import the MySQL connector, and create a connection with the username (pythoneverything), the password (python123), and host ().

import mysql.connector
conn = mysql.connector.connect(user='pythoneverything', password='python123', host='104.248.22.17', database='')

This created a connection. Let's try to create a database in MySQL:

cursor = conn.cursor()
cursor.execute("CREATE DATABASE development;")

This creates a database which is called development. We will now create table with two columns in this database, to check if everything works.

query = "CREATE TABLE development.test (name TEXT NOT NULL, created TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP);"
cursor.execute(query)

And retrieving all the table names, where we only expect to get 'test':

query = "SELECT TABLE_NAME FROM information_schema.tables where table_schema='development';"
cursor.execute(query)
for row in cursor.fetchall():
    print(row[0])

It seems to work! Now we have a database, which we can access with the following:

import mysql.connector
conn = mysql.connector.connect(
    user='pythoneverything', 
    password='python123', 
    host='104.248.22.17', 
    database=''
    )

Thanks for reading! And don't forget to navigate to Digital ocean and get started with your database! Oh and I'll help you save some time: The above credentials and IP address are destroyed, so connecting to my test database wouldn't work.
