From API documentation to Python code - The Postman method

2020-08-12 • By Ivo

How to generate code for APIs in seconds with Postman and Python

When working with Python, or any programming language for that matter, APIs are always a theme that has to be dealt with. My go to tool to check my API calls is Postman.

This short post will explain how to get from Postman to running Python code. Let's assume we would like to fetch the Fixer.io with their API (just like we did for our ETL series). To do so, we follow their documentation, and replicate the API calls with Postman. This will result in the following:

Now we have the API running in Postman. If we want to convert it to Python code, we only have press the 'code' button, search for Python and choose our prefered library. I always use the Requests library, but you are free to choose the http.client library instead.

We only have to copy-paste this into our Python script and we are good to go! 

import requests

url = "http://data.fixer.io/api/latest?access_key=API_KEY&base=EUR&symbols=USD"

payload = {}
headers = {
'Accept': 'application/json',
}

response = requests.request("GET", url, headers=headers, data = payload)

print(response.text.encode('utf8'))

Which gives us:

{
    "success": true,
    "timestamp": 1597253945,
    "base": "EUR",
    "date": "2020-08-12",
    "rates": {
        "USD": 1.17875
    }
}

Thanks for reading!
