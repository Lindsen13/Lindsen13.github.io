Executing Alteryx workflows and Apps with Python

2020-08-14 • By Ivo

Using the Alteryx API to execute workflows and apps on the Alteryx Gallery

For some of my clients, I am using Alteryx for all the data handling. It has some benefits over Python, since it's a drag-and-drop tool that is quite easy to master. So called workflows are easy to understand due to Alteryx's graphical interface. Every function or other action is represented by a tool, also called a macro.

One of the most powerful aspects of Alteryx, is their deployments option. They have a product that is called Alteryx Gallery, which comes with an API that allows us to execute workflows with a 'simple' GET-request.

Now personally I find the authentication method chosen by Alteryx a bit confusion. Therefore, I was looking for a simple python wrapper around this API.

Theamazingdp managed to make one! I came across his Github repository, and I would like to see if it works. He has some nice explanations on how to use his Gallery class object to make API calls based on your inputs.

To be able to use his package, we have to download it to a local folder and refer to this directory, rather than a library installed with pip. You can download his package through Github.

Let's get started with unpacking the zipped folder, and placing it into our working directory. Furthermore, we create a simple script.py file and start importing the package and defining the variables we need.

from AlteryxGalleryAPI import alteryx_gallery_api_v3_compatible as AlteryxAPI

client_key = {CLIENT_KEY}
client_secret = {CLIENT_SECRET}
gallery_url = 'https://alteryx.YOURALTERYXGALLERY.com/gallery/api/v1'  
workflow_id = {WORKFLOW_ID}

As you can see, we imported the alteryx_gallery_api_v3_compatible file as AlteryxAPI (just to create some more readable code down the line), to make sure we can use the Gallery class. Besides this, we define the client_key, client_secret, gallery_url and workflow_id. All of them are acquired from our Alteryx Gallery. Note: API credentials have to be enabled for Alteryx Gallery users (Artisans) before they appear in 'My Profile'.

After the import and the declaration of the variables, we can use alteryx_gallery_api_v3_compatible's Gallery class to create a connection.

con = AlteryxAPI.Gallery(gallery_url, client_key, client_secret)

This connection allows us call multiple methods, one of them is of interest of us. It is the execute_workflow() method. This allows us to parse a workflow ID from the Alteryx Gallery, and the function will make a API call to execute the workflow,.

output, output_content = con.execute_workflow(workflow_id)

we can see in alteryx_gallery_api_v3_compatible.py, that execute_workflow() returns two variables, output and output_content. Looking into the package, these 2 variables are returned as the following:

output, output_content = output, json.loads(output.content.decode("utf8"))
return output, output_content

To me this is a bit redundant, so I changed my copy of the package to make sure it only returns the output variable:

#output, output_content = output, json.loads(output.content.decode("utf8"))
return output

Which results in only one variable to be returned. I find it much cleaner. You are however more then welcome to stick with the original package.

output = con.execute_workflow(workflow_id)

Now we can see if the workflow has executed correctly:

if output.status_code == 200:
    print(f"workflow executed correctly at {output.json().get('createDate')}")

Which gives

workflow executed correctly at 2020-08-14T06:18:01.074374Z

Great! We now know how to execute any workflow with the Alteryx Gallery!

All together in one script:

from AlteryxGalleryAPI import alteryx_gallery_api_v3_compatible as AlteryxAPI

client_key = '8D8385BDCE90783dbd8717034398e5d1c48520cf1d621b89d48da84'
client_secret = 'db925d6ae9dbb93ffc3270e5cefc7ba15eca2411d7aa97ac83cab3094dc47c92'
