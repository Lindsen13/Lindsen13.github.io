Build your own Application - Deployment on Heroku

2020-09-23 • By Ivo

Using Flask and Heroku to create a simple yet amazing webpage for our application

See the final result here

This is the fourth posts for my series. If you haven't read my first, second and third post, please do so here, here and here. I will show you how you can make a custom application that includes predictive modelling! The creation of this small SAAS app is split up in the following parts:

  • part 1: Scraping data
  • part 2: Predictive modelling
  • part 3: Web interface
  • part 4: Deployment

Lets deploy our application! It is a web interface that takes several characteristics of a car. Based on the characteristics, a prediction is made for the value of the car.

Lets get started on deploying our application to heroku

Lets continue on our current directory

# Navigate into the folder
cd MyAmazingApplication
# Activate virtual Environment
source env/bin/activate

We have to install gunicorn, which allows us to create a simple web-server:

pip3 install gunicorn

We have to create a requirements.txt file with all our libraries. To retrieve a list of our libraries (and their respective versions), we can run:

pip3 freeze

If you have followed along in this series, it should like something like this:

We can simply copy and paste them into a requirements.txt file, but this can also be done in the terminal directly:

pip3 freeze > requirements.txt

Furthermore, we need to create a Procfile with a single line stating 

web: gunicorn wsgi:app

Again, copy-paste should do the trick. Alternatively, run:

echo "web: gunicorn wsgi:app" >> Procfile

Now our project is ready to be deployed. We will use Heroku's CLI for this. Heroku has a nice guide on how to install the CLI, which I will briefly run to:

brew tap heroku/brew && brew install heroku

And that's it! Homebrew (if you have it on your pc) will install the CLI for Heroku. We can now login:

heroku login

After providing a password, we can create a app:

heroku create app_name_here --buildpack heroku/python

Note that app_name_here will be part of the URL you'll get from heroku. Mine is https://car-price-app.herokuapp.com/, which means I used car_price_app here. Besides the app name, we also specify the buildpack. We are interested in deploying a python application, therefore we specify the heroku/python buildpack.

Great! To deploy this to Heroku, we will initiate a GIT repository, add all our code and commit it.

git init .
git add .
git commit -m "first commit, getting ready to deploy to Heroku!"

Ready to push to heroku?

git push origin master

And that should be it! You will now be able to find your application at https://app_name_here.herokuapp.com/.

All code together:

python3 -m venv env
source env/bin/activate
pip3 install dependencies
pip3 freeze > requirements.txt
echo "web: gunicorn wsgi:app" >> Procfile
heroku login
heroku create app_name_here --buildpack heroku/python
git init
git add .
git commit -m "first commit"
git push origin master

This is the final post in our 4-part series. Ofcourse, the application is extremely simple, and can't really be used to make any decisions on the outcome. The idea behind this application was to show solely how something like this would work.

One improvement (which would be highly needed!), would be to change the forecasting method. Right now, we insert all possible variables in the model, and run it without checking the performance. Doing so, would surely improve the predicitve power of the model.

Next to this, the application is not storing the data it has processed, nor is it asking for any user input. Collecting this could help improving the prediction in the future, and could also help to monetize the application (think about collecting email addresses!).

These are just 2 examples of obvious changes, where there are many more. Think about adding Google Analytics to track the visitors, or some more pages to describe what the tool is about.
