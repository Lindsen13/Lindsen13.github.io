Make your own API with Flask and Python, with Authentication!

2020-07-26 • By Ivo

Use Flask with HTTPBasicAuth to retrieve and query a list of items from

Today I'd like to share a simple method on how to create an API with authentication! We are going to use Flask and Flask-HTTPAuth to achieve this.

Let's initiate a directory with the right dependencies:

# create a directory called 'flask_api'
mkdir flask_api
# Navigate into the directory
cd flask_api
# Create a virtual environment
python3 -m venv env
# Activate the virtual environment
source env/bin/activate
# Install the 'Flask' package.
pip install Flask
# Install the 'Flask-HTTPAuth' package.
pip install Flask-HTTPAuth
# Create a python file for the API
touch app.py

Let's open the app.py file and get started! First we import the libraries which we need.

from flask import Flask, jsonify, make_response
from flask_httpauth import HTTPBasicAuth

The Flask class from flask helps us to create a Web Server Gateway Interface application, the make_response() function helps us to create a HTTP 403 error, which means Unauthorized and Jsonify from Flask helps us to return a JSON object. The last import is HTTPBasicAuth from flask_httpauth. This extension helps us to handle HTTP authentication.

app = Flask(__name__)
auth = HTTPBasicAuth()

Just as any normal flask application, we assign the Flask class to the app variable. Furthermore, we assign the HTTPBasicAuth() function to the auth variable. This allows us to generate routes including authentication. Lets start out with adding a function to the auth object that checks the username and password for a user.

@auth.get_password
def get_password(username):
    if username == 'username':
        return 'password'
    return None

We use @auth.get_password to add to the auth object using Python's '@'-decorator. We create the get_password()* function, which checks the username and password combination. For an API in production, you will most likely use a database with hashed passwords, but for this example we return the user if the password is equal to password and the username is equal to username.

Next up, let's make a function that handled unauthorized requests.

@auth.error_handler
def unauthorized():
    return make_response(jsonify({'error': 'Unauthorized access'}), 401)

This function returns a error message in JSON format with the make_response function. We can use these two functions to make a really simple API that allows us to populate a list of things, and retrieve the list of things.

data = []

@app.route('/', methods = ['GET'], defaults={'info':None})
@app.route('/<info>', methods = ['POST','GET'])
@auth.login_required
def input_data(info):
    global data
    if info:
        data.append(info)
    return jsonify(data)

The code above allows us to use two endpoints: / and /<info>. The first endpoint returns the current list, and the second endpoint adds the variable <info> to the list. Let's run the script and see it in action!

python app.py

Let's navigate to http://localhost:5000/. This should return:

After putting in the right credentials (username for username and password for password), we will see an empty list ([ ]) in the browser. If we want to add an item, for example apple, we can do that by replacing <info> for our API endpoint. Visiting http://localhost:5000/apple will add apple to the list of items.

Note that we store the information on the current session in your terminal. If you would end the session and start a new session with 

python app.py

You will see that the list has disapeared. Obviously you want to make sure we are directly querying on a database, rather then just appending and reading from a local variable.
